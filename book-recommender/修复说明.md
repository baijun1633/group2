# 代码修复说明

## 修复的问题汇总

### 🔴 严重 Bug（6个，全部修复）

#### 1. 图书节点属性名不匹配
- **问题**：`build_kg.py` 创建节点用 `book_id`，但查询代码全用 `bookId`
- **修复**：统一为 `bookId`（驼峰命名）
- **影响文件**：`script/build_kg.py`

#### 2. 关系类型大小写不一致
- **问题**：`build_kg.py` 用大写（`WRITTEN_BY`），查询用小写（`written_by`）
- **修复**：统一为小写关系类型
- **影响文件**：`script/build_kg.py`
- **涉及关系**：`written_by`、`published_by`、`belongs_to`、`tagged_as`、`belongs_to_parent`

#### 3. settings.py 缩进语法错误
- **问题**：MySQL 配置注释前多了一个空格，导致 `IndentationError`
- **修复**：修正缩进，整理配置格式
- **影响文件**：`app/config/settings.py`

#### 4. kg_service.py 深度参数化不支持
- **问题**：Neo4j 不支持 `[*1..$depth]` 这种参数化关系深度
- **修复**：改用 f-string 拼接，配合 API 层的参数校验保证安全
- **影响文件**：`app/service/kg_service.py`

#### 5. kg_reasoning.py UNION 字段名不一致
- **问题**：三个 UNION 子查询返回的列名不一致（`hops` vs `min_hops`）
- **修复**：统一子查询返回列名，UNION 后再做聚合计算
- **影响文件**：`app/algorithm/kg_reasoning.py`

#### 6. requirements.txt 缺少 pymysql
- **问题**：代码中 `import pymysql`，但依赖清单里没有
- **修复**：添加 `pymysql==1.1.0`
- **影响文件**：`requirements.txt`

---

### 🟡 中等问题（4个，全部修复）

#### 7. MySQL 连接无重连机制
- **问题**：全局单例连接，断开后不会自动重连
- **修复**：添加 `_ensure_connection()` 方法，查询前检查连接有效性，失效自动重连
- **影响文件**：`app/database/mysql_client.py`

#### 8. Neo4j 驱动非线程安全
- **问题**：`_ensure_driver()` 没有加锁，多线程并发可能创建多个实例
- **修复**：添加双重检查锁（DCL），保证线程安全
- **影响文件**：`app/database/neo4j_client.py`

#### 9. FastAPI 事件处理器已废弃
- **问题**：`@app.on_event("startup")` 和 `@app.on_event("shutdown")` 已过时
- **修复**：改用 `lifespan` 上下文管理器
- **影响文件**：`main.py`

#### 10. 缺少日志系统
- **问题**：全用 `print()`，无法分级、无法持久化
- **修复**：引入 `logging` 模块，配置统一日志格式
- **影响文件**：`main.py`、`app/database/*.py`、`app/service/*.py`、`script/build_kg.py`

## 使用说明

1. 安装依赖：
   ```bash
   pip install -r requirements.txt
   ```

2. 配置 `.env` 文件中的数据库连接信息

3. 构建知识图谱（首次使用）：
   ```bash
   python script/build_kg.py
   ```

4. 启动服务：
   ```bash
   python main.py
   ```

5. 访问接口文档：
   - Swagger UI: http://localhost:8001/docs
   - ReDoc: http://localhost:8001/redoc

## 注意事项

⚠️ **重要**：如果你之前已经用旧版 `build_kg.py` 构建过知识图谱，必须重新运行构建脚本，因为属性名和关系类型都做了统一修正。
